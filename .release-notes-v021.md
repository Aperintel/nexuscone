Docs-only patch release. No code changes between v0.2.0 and v0.2.1.

## What changed

This release exists to refresh the README that PyPI displays on the project page. The v0.2.0 sdist embedded the README as it stood at build time on 2026-05-25, before the post-release README polish that landed in commit `20c73a9`. PyPI versions are immutable by design, so a fresh patch release was the cleanest way to surface the corrected README on the live project page.

Concrete README updates included:

- Metacarpal positioning refreshed to the accountability-first framing (governance-grade autonomous-agent operating system, both personal and enterprise on one auditable spine).
- Aperintel governance product renamed from Nexus to Hyperaxis, with the Discover, Govern, Sign three-pillar framing replacing the previous feature list.
- Regulatory framing tightened to FCA, EU AI Act Article 12, and NHS DSPT to match the current product trajectory.
- New Roadmap section covering v0.3.x AWS-native deployment (ECR image, Lambda Layer, CloudFormation and Terraform modules, optional DynamoDB and Aurora PostgreSQL ledger backends, AWS KMS adapter, CloudTrail anchor adapter) and v0.4.x Merkle inclusion proofs plus selective disclosure with redaction proofs.

## Install

```bash
pip install --upgrade nexuscone
```

Will resolve to v0.2.1.

## Compatibility

v0.2.1 is byte-equivalent to v0.2.0 in terms of installed Python code. All v0.2.0 features work identically; no migration is required. If you have v0.2.0 installed, upgrading is optional from a functional perspective.
