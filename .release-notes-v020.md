This release moves Nexuscone from a self-trust audit chain to a publicly verifiable one. Every chain head can now be anchored, in parallel, to Bitcoin's blockchain via OpenTimestamps and to a regulator-friendly RFC 3161 Time-Stamp Authority. A consortium of named witnesses can also co-sign the chain head between Bitcoin confirmations.

## Headline features

- **OpenTimestamps anchoring to Bitcoin** via free public calendar servers. No wallet, no node, no token, no transaction fee.
- **RFC 3161 TSA secondary anchor** signs the same chain head immediately, giving a regulator-verifiable token the moment it arrives.
- **Witness-Inclusion Beacon (consortium tier)** lets a named set of witnesses sign the chain head between Bitcoin confirmations, closing the gap between submission and proof-of-work finality.
- **Background poller** (`upgrade_pending_anchors`) asks calendars to upgrade OpenTimestamps proofs in place once Bitcoin mines the relevant block, and records the confirmed block height.
- **CLI verification surfaces:** `nexuscone-verify --check-anchors`, `--check-tst`, `--upgrade-pending`, `--print-anchor`.
- **End-to-end test** drives the full anchoring pipeline against 1500 entries with mocked OpenTimestamps and a synthetic Bitcoin confirmation.

## Trust model

Three integrity tiers now stack on top of each other:

1. **Chain integrity** (database only). Walking the chain recomputes every hash; any tamper produces a mismatch.
2. **Ed25519 signatures** (database plus public key). Proves the row could only have been produced by the matching private signing key.
3. **Anchored timestamps** (database plus proof blob plus Bitcoin block headers or TSA certificate). Proves the chain head existed at a specific point in time, verifiable without trusting Nexuscone, Aperintel, or any single party.

The lower levels are always available; the higher levels add stronger guarantees at the cost of a few more dependencies on external infrastructure.

## Install

```bash
pip install nexuscone
```

OpenTimestamps and RFC 3161 client libraries are required dependencies in v0.2.0. The optional Ed25519 signing layer remains available as `pip install "nexuscone[signing]"`.

## Quick start with anchoring

```python
import asyncio
from nexuscone import Ledger
from nexuscone.anchor_schedule import AnchorSchedule


async def main() -> None:
    schedule = AnchorSchedule(every_n_entries=1000, every_m_minutes=60, enabled=True)
    async with Ledger("data/audit.db", anchor_schedule=schedule) as ledger:
        await ledger.log(actor="api", action="request", payload={"id": "r-001"})
        anchor = await ledger.anchor()
        if anchor is not None:
            print(f"anchored {anchor.chain_head_hash[:16]} at {anchor.submitted_at}")


asyncio.run(main())
```

## Quality gates

99 pytest cases pass. Ruff clean. Mypy strict clean across 11 source files. CI runs on Python 3.10, 3.11, 3.12, and 3.13.

## Compatibility

v0.1.0 audit chains continue to verify under their original six-field hash formula because the verifier dispatches by `format_version`. New v0.2.0 chains use a seven-field formula adding `event_type`.

## Roadmap

v0.3.x targets AWS-native deployment: public ECR container image, AWS Lambda Layer, CloudFormation and Terraform reference modules, optional DynamoDB and Aurora PostgreSQL ledger backends, an AWS KMS adapter for signing keys, and a CloudTrail anchor adapter that pulls CloudTrail events into the Nexuscone tamper-evident chain. v0.4.x targets Merkle inclusion proofs and selective disclosure with redaction proofs.
